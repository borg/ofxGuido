5 Exact Positioning and Formatting of Basic Notation Elements

In this section, the mechanisms for the exact formatting and positioning of basic notation elements are presented. In most cases, the corresponding Basic GUIDO tags are extended with new parameters; however, some new tags are introduced while extensions of some of the Basic GUIDO tags are discussed in other sections (such as \text).

5.1 Dynamic markings

5.1.1 \intens, \i

This tag indicates a dynamic marking and associates it with a relative intensity value. The \intens tag is associated with the metrical position of the note immediately following it and carries to the next dynamic marking. \i and \intens are synonymous.

Parameters (in the order of their default positions):

  • m (string; obligatory) - dynamic marking as displayed in a score (like "f", "p", "mf", "fp", etc.).
  • i (float; optional) - relative intensity, a unit-less floating point number between 0.0 (minimal intensity) and 1.0 (maximal intensity).
    Default value: unspecified. If i is not specified, GUIDO clients may try to infer a value from the m parameter or use an application specific default value.
  • dx (float; optional) - horizontal offset between center of leftmost character of the marking and center of the following note head; if no units are specified, the units defined by the \units tag are used.

    • Default value: 0
  • dy (float; optional) - vertical distance between center of the marking and the bottom staff line; if no units are specified, the units defined by the \units tag are used.
    Default value: unspecified; if no dy value is specified, the vertical position should be automatically determined such that the marking appears below the staff and collisions are avoided.
  • size (float; optional) - fontsize in which the marking is displayed; if no units are specified, the units defined by the \units tag are used.

    • Default value: 12pt(??)

Range: none

Remarks: (none) Examples:

    \intens<m="f", dx=-2.0, dy=2>
    \intens<m="ff", i=0.99, dy=-5>
      ... 

5.1.2 \cresc, \dim, \decresc

These tags are used to realise crescendos and diminuendos (or decrescendos); \dim and \decresc are synonymous. For all three tags, two graphical styles exist: one, where the marking is displayed as a wedge, and a second, where it is displayed as text plus a dashed line indicating the range. The beginning dynamic marking (and intensity) is taken from the last \intens setting (or unspecified, if no \intens occurred before in the current voice); the ending dynamic marking (and intensity) can be specified with these tags.

Parameters (in the order of their default positions):

  • m (string; optional) - dynamic marking (string) as displayed in a score (like "f", "p", "mf", "fp", etc.).
    Default value: unspecified
  • i (float; optional) - relative intensity at the end of the cresc/dim/decresc; a unit-less floating point number between 0.0 (minimal intensity) and 1.0 (maximal intensity). If this parameter is not specified,
    Default value: unspecified; if m is specified, GUIDO clients may use this for inferring a i value from or use an application specific default value.
  • style (string; optional) - "<" for \cresc or ">" for \dim, \decresc for using wedge symbols, "-" for textual markings and dashed lines.
    Default value: "<" for \cresc and ">" for \dim and \decresc.
  • dx (float; optional) - horizontal offset between center of last note head in range and center of first character of the dynamic marking; if no units are specified, the units defined by the \units tag are used.

    • Default value: 0
  • dy (float; optional; used only for "-" style) - vertical distance between center of the dynamic marking and the bottom staff line; if no units are specified, the units defined by the \units tag are used.
    Default value: If no dy value is specified, the vertical position should be automatically determined such that the marking appears below the staff and collisions are avoided.
  • dx1 (float; optional) - for "<" and ">" styles: horizontal offset between center of first note head in range and left end of wedge; for "-" style: horizontal offset between center of first note head in range and center of first character of the string "cresc.", "dim.", or "decresc."; if no units are specified, the units defined by the \units tag are used.

    • Default value: 0
  • dy1 (float; optional) - for "<" and ">" styles: vertical distance between center of left end of wedge and bottom staff line; for "-" style: vertical distance between center of dashed line and and bottom staff line; if no units are specified, the units defined by the \units tag are used.
    Default value: If no dy1 value is specified, the vertical position should be automatically determined such that the wedge appears below the staff and collisions are avoided.
  • dx2 (float; optional) - for "<" and ">" styles: horizontal offset between center of last note head in range and right end of wedge symbol; for "-" style: horizontal offset between center of last note head in range and right end of dashed line; if no units are specified, the units defined by the \units tag are used.

    • Default value: 0
  • dy2 (float; optional) - for "<" and ">" styles: vertical distance between center of right end of wedge and bottom staff line; for "-" style: vertical distance between center of right end of dashed line and bottom staff line; if no units are specified, the units defined by the \units tag are used.
    Default value: unspecified. If no dy2 value is specified, the vertical position is automatically determined such that the wedge appears below the staff and collisions are avoided.
  • h (float; optional; used only for "<" and ">" styles) height of the wedge, measured on its open end. If no units are specified, the units defined by the \units tag are used.

    • Default value: 3hs(??)
  • msize (float; optional) - fontsize for the dynamic marking specified via the m parameter. If no units are specified, the units defined by the \units tag are used.

    • Default value: 12pt(??)
  • tsize (float; optional; used only for "-" style) - fontsize for the string "cresc", "dim", or "decresc", indicating the dynamic marking; if no units are specified, the units defined by the \units tag are used.

    • Default value: 12pt(??)

Range: oligatory

Remarks:

  • For crescendos and diminuendos the appropriate tags and parameters should be used, using \cresc for representing diminuendos or \dim for crescendos should be avoided.
  • For longer crescendo or diminuendo sections the \crescBegin and \crescEnd, \dimBegin and \dimEnd, or \decrescBegin and \decrescEnd tag combinations should be used (see below); this applies also for situations, where the crescendo or diminuendo section overlaps with other tagged ranges such as slurs.

Examples:

    \cresc<m="p", "<", dx1=0, dy1=2,dy2=-2>
    \dim<"-",dy=3>
    ...

5.1.3\crescBegin, \dimBegin, \decrescBegin

These tags indicate the beginning of a crescendo/diminuendo section which ends at the next \crescEnd/\dimEnd tag; the crescendo starts from the current dynamic marking. Parameters (in the order of their default positions):
  • style (string; optional) - same as for \cresc, \dim, \decresc.
    Default value: "<" for \cresc and ">" for \dim and \decresc.
  • dx (float; optional) - horizontal offset between center of last note head in range and center of first character of the dynamic marking; if no units are specified, the units defined by the \units tag are used.

    • Default value: 0
  • dy (float; optional; used only for "-" style) - vertical distance between center of the string "cresc", "dim", or "decresc", indicating the dynamic marking, and the bottom staff line; if no units are specified, the units defined by the \units tag are used.
    Default value: unspecified. If no dy value is specified, the vertical position should be automatically determined such that the marking appears below the staff and collisions are avoided.
  • dx1 (float; optional) - for "<" and ">" styles: horizontal offset between center of first note head in range and left end of wedge; for "-" style: horizontal offset between center of first note head in range and center of first character of the string "cresc.", "dim.", or "decresc."; if no units are specified, the units defined by the \units tag are used.

    • Default value: 0
  • dy1 (float; optional) - for "<" and ">" styles: vertical distance between center of left end of wedge and bottom staff line; for "-" style: vertical distance between center of dashed line and and bottom staff line; if no units are specified, the units defined by the \units tag are used.
    Default value: If no dy1 value is specified, the vertical position should be automatically determined such that the wedge appears below the staff and collisions are avoided.
  • tsize (float; optional; used only for "-" style) - fontsize for the string "cresc", "dim", or "decresc", indicating the dynamic marking; if no units are specified, the units defined by the \units tag are used.

    • Default value: 12pt(??)

Range: none

Remarks: (none)

Examples:

    \crescBegin<"<",dx=1,dy=-2>
    \dimBegin<"-",size=2>
    ...

These tags indicate the end of a crescendo\diminuendo section which began at the last \crescBegin/\dimBegin tag. The style attribute (which determines which parameter these tags accept) is specified with the corresponding \crescBegin / \dimBegin / \decrescBegin tags.

Parameters (in the order of their default positions):

  • m (string; optional) - dynamic marking (string) as displayed in a score (like "f", "p", "mf", "fp", etc.).
    Default value: unspecified
  • i (float; optional) - relative intensity at the end of the cresc/dim/decresc; a unit-less floating point number between 0.0 (minimal intensity) and 1.0 (maximal intensity). If this parameter is not specified,
    Default value: unspecified; if m is specified, GUIDO clients may use this for inferring a i value from or use an application specific default value.
  • dx2 (float; optional) - for "<" and ">" styles: horizontal offset between center of last note head in range and right end of wedge symbol; for "-" style: horizontal offset between center of last note head in range and right end of dashed line; if no units are specified, the units defined by the \units tag are used.

    • Default value: 0
  • dy2 (float; optional) - for "<" and ">" styles: vertical distance between center of right end of wedge and bottom staff line; for "-" style: vertical distance between center of right end of dashed line and bottom staff line; if no units are specified, the units defined by the \units tag are used.
    Default value: unspecified. If no dy2 value is specified, the vertical position is automatically determined such that the wedge appears below the staff and collisions are avoided.
  • h (float; optional; used only for "<" and ">" styles) height of the wedge, measured on its open end. If no units are specified, the units defined by the \units tag are used.

    • Default value: 3hs(??)
  • msize (float; optional) - fontsize for the dynamic marking specified via the m parameter. If no units are specified, the units defined by the \units tag are used.

    • Default value: 12pt(??)

Range: none

Remarks: (none)

5.2 Tempo indications

5.2.1 \tempo

This tagis used to indicate tempo markings; at the same time it allows to specify the absolute tempo for playback.

Parameters (in the order of their default positions):

  • m (string; obligatory) - tempo marking as it appears in the score, e.g. "Allegro", "Andante", "slower", "gemächlich".
  • bpm (string; optional) - absolute tempo; a string of the form "x/y=n" where x/y is a duration value like 1/4 or 3/8 and n an integer number which indicates the number of these duration values per minute.

    • Default value: unspecified; GUIDO clients may use the m value for inferring an absolute tempo or use an application specific default value.
  • dx - horizontal offset between center of first note head after the tag and center of first character of the tempo indication m; if no units are specified, the units defined by the \units tag are used.

    • Default value: 0
  • dy - vertical distance between the top staff line and the center of the tempo indication m; if no units are specified, the units defined by the \units tag are used.
    Default value: If no dy value is specified, the vertical position shoule be automatically determined such that the marking appears above the staff and collisions are avoided.
  • size - fontsize for the tempo marking specified via the m parameter. If no units are specified, the units defined by the \units tag are used.

    • default value: 12pt (??)

Range: none

Remarks:

  • The \tempo tag is associated with the metrical position of the note immediately following it and carries to the next tempo indication.

Examples:

    \tempo<m="presto",dx=1,dy=-5>
    \tempo<m="andante",bpm="1/8=200",dx=8>
    ...

5.2.2 \accel,\rit

These tags indicates an accelerando/ritardando; the beginning tempo marking (and absolute tempo) is taken from the last \tempo setting (or unspecified, if no \tempo occurred before in the current voice); the ending tempo marking (and absolute tempo) can be specified with these tags. The accelerando/ritardando is graphically marked by the text "accel."/"rit." and a horizontal dashed line indicating the range.

Parameters (in the order of their default positions):

  • m (string; optional) - tempo marking as applied to the last note in the range.
    Default value: ""
  • bpm (string; optional) - absolute tempo reached at the beginning of the last note in the range; a string of the form "x/y=n" where x/y is a duration value like 1/4 or 3/8 and n an integer number which indicates the number of these duration values per minute.
    Default value: unspecified; if m is specified, GUIDO clients may use this for inferring an absolute tempo or use an application specific default value.
  • dx (float; optional) - horizontal offset between center of first note head in range and center of first character of the tempo indication m; if no units are specified, the units defined by the \units tag are used.

    • Default value: 0
  • dy (float; optional) - vertical distance between center of the tempo indication m and the top staff line; if no units are specified, the units defined by the \units tag are used.
    Default value: unspeciefied. If no dy value is specified, the vertical position should be automatically determined such that the marking appears above the staff and collisions are avoided.
  • msize (float; optional) - fontsize for the tempo marking specified via the m parameter. If no units are specified, the units defined by the \units tag are used.

    • Default value: 12pt (??)
  • tsize (float; optional) - fontsize for the string "accel."/"rit." indicating the accelerando/ritardando. If no units are specified, the units defined by the \units tag are used.

    • Default value: 12pt (??)

Range: obligatory

Remarks:

  • For accelerandos and ritardandos the appropriate tags and parameters should be used, using \accel for representing ritardandos or \rit for accelerandos should be avoided.
  • For longer accelerando or ritardando sections the \accelBegin and \accelEnd or the \ritBegin and \ritEnd tag combinations should be used; this applies also for situations, where the accelerando or ritardando section overlaps with other tagged ranges such as slurs.

5.3 \instr

5.3.1 Instrument indications

This tag sets defines the instrument for the current voice to and associates it with an an instrument identifier used for playback.

Parameters (in the order of their default positions):

  • m (string; obligatory) - instrument marking, appears at the beginning of the score for \instr tags which appear before the first note of a voice, above the note following it, if \instr appears within a voice. It is recommended that the standard names or abbreviations commonly used in scores be used.
  • id (string; optional) - instrument identifier used for specifying a playback instrument. MIDI instruments can be accessed by values of the form "GM n" (for general MIDI instrument number n), where n is an integer between 1 and 128.
    Default value: unspecified. GUIDO clients may try to infer an playback instrument from the m parameter or use an application specific default value.
  • dx (float; optional) - when tag appears before first note of voice (case 1): horizontal displacement between left margin of system and right margin of the bounding box of instrument marking m; otherwise (case 2): horizontal distance between the center of the head of the next note after the tag and the center of first character of the instrument indication m; if no units are specified, the units defined by the \units tag are used.

    • Default value: 5hs (case 1) / 0 (case 2)

    Default value: 0 (case 1) / unspecified (case 2); If in case 2, no dy value is specified, GUIDO notation clients should automatically determine the vertical position such that the marking appears above the staff and collisions are avoided.
  • size (float; optional) - fontsize for the instrument marking specified via the m parameter. If no units are specified, the units defined by the \units tag are used.

5.4 Slurs and ties

5.4.1 \slur

This tag slurs notes together. For slurred chords, only one slur is graphically represented. Graphically, a model with three control points is assumed, the outermost of which are located at either end of the slur; the middle control point is located between these on the slur at the point of maximal distance between the slur and the straight line connecting the two centers of leftmost and rightmost note heads in the range.

Parameters (in the order of their default positions):

    dx1 (float; optional) - horizontal distance between the center of the first note head in the range and the left end of the tie; if no units are specified, the units defined by the \units tag are used.
    Default value: 2hs
  • dy1 (float; optional) - vertical distance between the center of the leftmost note head and the left end of the slur; if no units are specified, the units defined by the \units tag are used.
    Default value: 1hs
  • dx2 (float; optional) - horizontal distance between the center of the last note head in the range and the right end of the tie; if no units are specified, the units defined by the \units tag are used.
    Default value: -2hs
  • dy2 (float; optional) - vertical distance between center of right note head in the range and the right end of the slur; if no units are specified, the units defined by the \units tag are used.
    Default value: 1hs
  • r3 (float; optional) - relative horizontal position of the middle control point between the centers of the two outmost note heads. The value is between 0.0 (leftmost) and 1.0 (rightmost); the value has no unit.
    Default value: (determined by the implementation, dependent on the positions of the notes in the range)
  • h (float; optional) - vertical distance between the point on the straight line connecting the centers of the leftmost and rightmost slurred notes determined by the r3 parameter and the middle control point; if no units are specified, the units defined by the \units tag are used.
    Default value: (determined by the implementation, dependent on the positions of the notes in the range)

Range: obligatory

Remarks:

  • \slur should not be used to represent tied notes, for which the \tie tag is to be used.
Examples:

5.4.3 \tie

This tag ties notes together; it is only to be used for two or more notes with identical pitch. For tied chords, the number of voices in each chord and all pitches have to be be identical; in this case, the voices in all voices are tied, respectively.

Parameters (in the order of their default positions):

    dx1 (float; optional) - horizontal distance between the center of the first note head in the range and the left end of the tie; if no units are specified, the units defined by the \units tag are used.
    Default value: 2hs
  • dy1 (float; optional) - vertical distance between center of the note heads (note that the notes have the same pitch) and the ends of the tie; if no units are specified, the units defined by the \units tag are used.
    Default value: 1hs
  • dx2 (float; optional) - horizontal distance between the center of the last note head in the range and the right end of the tie; if no units are specified, the units defined by the \units tag are used.
    Default value: -2hs
  • r3 (float; optional) - relative horizontal position of the middle control point (highest point of the tie) between the centers of the two outmost note heads. The value is between 0.0 (leftmost) and 1.0 (rightmost); the value has no unit.
    Default value: 0.5
  • h (float; optional) - vertical distance between center of the note heads and the highest point of the tie; if no units are specified, the units defined by the \units tag are used.
    default value: 2.5hs

Range: obligatory

Remarks:

  • If more than two notes appear in the range, ties are displayed between each pair of neigbouring notes.
  • All notes in the range have to have identical pitch.
  • Ties are neither identical with nor equivalent to slurs between notes with identical pitches. Whereas a slur indicates a grouping, a tie musically merges the notes in its range.
  • Tied note groups are neither identical with nor equivalent to merged groups. While a merged group corresponds to a single note which is notated as one note symbol, a tied group is notated as a number of notes connected by ties.

5.6 Trills and Ornaments

5.6.1 \trill, \mord, \turn

These tags indicate trills/mordents/turns; they are applied to a range containing one or more chords, where the first voice of each chord specifies the principal note and the other voices auxiliary notes.

  \trill<speed,dx,dy,size>

Parameters (in the order of their default positions):

  • speed (string; optional) - string of the form "m/n" specifies the ornament to be executed as a sequence of m/n notes. For trills, alternating notes span the full duration.
    Default value: (not specified, depends on implementation of application.) dx (float; optional) - horizontal distance between center of leftmost dot and center of note head, in halfspaces of the current system.

    • Default value: 2.0
  • dy (float; optional) - vertical distance between center of dot and center of note head, in halfspaces of the current system.

    • Default value: none, see Semantics.
  • size (float; optioal) - size of the symbol, as a factor relative to the size of the note head. Cannot be specified with a unit.

    • Default value: 1.0

Range: obligatory

Semantics:

  • If no dy value is specified, the vertical position is automatically determined such that it is above the staff and collisions are avoided.
Remarks:
  • For trills and mordents, each chord should have exactly two voices which specifiy the main and auxiliary note, respectively. Upward and downward trills as well as standard and inverse mordents can be realised by specifying these notes accordingly.
  • For turns, each chord should have exactly three voices which specifiy the main, the first and the second auxiliary note, respectively. Standard and inverse turns can be realised by specifying these notes accordingly.

3.3.18 \key

This tag indicates a key

\key<sig,size1,...,sizeN>

\key<name,size1,...,sizeN>

Parameters (in the order of their default positions):
Call1:

  • name - of the form "n", "n#" or "n&" for any note name n. Uppercase and lowercase letters are used to denote major and minor modes, respectively.
  • sizeX - fontsize of accidental number X (counting from left to right), measured in haplfspaces of the current system

    • default value: size(X-1)
Call2:
  • sig - sets key signature to i sharps, if i is positive, to |i| flats, if i is negative. For i=0, the key signature contains no sharps or flats.
  • sizeX -  fontsize of accidental number X (counting from left to right), measured in haplfspaces of the current system.

    • default value: size(X-1)
Range: optional

Semantics:

Remarks:

  • The \key tag can appear at the beginning or in the middle of a voice, the latter indicating a change in meter; Within one piece, one of the two formats of the key-tag should be used consistently.

3.3.19 \oct

This tag indicates the beginning of a section which is transposed upwards.

\oct<i,dy,size>
  \oct<i,dx1,dy1,dx2,dy2,size>

Parameters (in the order of their default positions):

Call1:

  • transp - number of octaves for the transposition
  • dy - vertical distance between center of octave marking and the top staffline, in halfspaces of the current system.

    • default value: none, see Semantics.
  • size - fontsize, measured in halfpaces of the current system.

    •  
Call2:
  • transp - number of octaves for the transposition
  • dx1 - horizontal distance between center of rightmost note head in the group and the right end of the octave marking measured in halfspaces of the current system.

    • default value: 0
  • dy1 - vertical distance between and the top staffline and the center of the octave marking measured in halfspaces of the current system.

    • default value: none, see Semantics.
  • dx2 - horizontal distance between center of rightmost note head in the group and the right end of the octave marking measured in halfspaces of the current system.

    • default value: 0
  • dy2 - vertical distance between and the top staffline and the center of the octave marking measured in halfspaces of the current system.

    • default value: none, see Semantics.
  • size - fontsize, measured in halfpaces of the current system.
Range: optional

Semantics:

  • If no dy/dy1/dy2 value is specified, the vertical position is automatically determined such that it is above the staff and collisions are avoided.
Remarks:
  • Octave transposition does affect the notated pitches such that "\oct<+1>(c1)" is pitch-equivalent to "c2".
  • Negative numbers i can be used to indicate downward transpositions.
  • \oct<+1> corresponds to "8va" in standard notation, \oct<-1> to "8va bassa", \oct<+2> to "15ma" etc.
